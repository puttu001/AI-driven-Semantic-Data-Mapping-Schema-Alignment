# Artifacts Directory

This directory contains all output files generated by the CDM mapping pipeline.

## Directory Structure

```
artifacts/
├── mapped/          # Successfully mapped CDM columns
├── unmapped/        # Columns that could not be mapped
└── validation/      # Validation ground truth files
```

## Subdirectories

### mapped/
Contains CSV files with final CDM mappings that were accepted, modified, or auto-rejected during the workflow.

**File Format:** `Final_CDM_Mappings_langgraph_<model_type>_<timestamp>.csv`

**Columns:**
- CSV source columns (App Table name, App Column Name, etc.)
- Mapped CDM columns (CDM Parent Mapped, CDM Column Mapped, etc.)
- Top candidates and reasoning
- LLM recommendations for new terms

### unmapped/
Contains CSV files with columns that could not be successfully mapped to the CDM glossary.

**File Format:** `Unmapped_Columns_langgraph_<model_type>_<timestamp>.csv`

**Columns:**
- CSV source columns
- Top 3 CDM column candidates
- Top 3 CDM parent candidates
- Reason for being unmapped

### validation/
Contains ground truth validation files used to assess mapping accuracy.

**Expected File:** `Validation_Sheet_Iteration Third.csv`

## Usage

All output files are automatically saved to their respective subdirectories when running the main pipeline:

```bash
python main.py
```

The pipeline will:
1. Process input files from `Inputs/` directory
2. Generate mappings through the interactive workflow
3. Save results to `artifacts/mapped/` and `artifacts/unmapped/`
4. Optionally validate against files in `artifacts/validation/`

## File Retention

Output files are retained with timestamps to maintain a history of pipeline executions. You can manually clean up old files or implement automated cleanup as needed.
